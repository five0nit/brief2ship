# Build Request Template

Build me a [thing] for [user] that does [main job].

It should feel like [reference / product / vibe].

Constraints:
- platform/output: [web / mobile / internal tool / automation / Markdown / PDF / HTML / slides]
- speed vs polish: [fast / balanced / polished]
- stack preferences: [optional]
- auth/integrations/deployment: [optional]
- anything to avoid: [optional]

For reports/documents, also specify:
- reader / audience: [who will read it]
- decision or action supported: [what they should decide/do]
- source material: [paths/links/data]
- tone and depth: [executive / technical / client-facing / internal]
